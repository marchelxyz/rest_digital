# Интеграция iiko с Rest Digital

Официальная документация iiko: [ru.iiko.help](https://ru.iiko.help), [api.iiko.ru](https://api.iiko.ru)

## Что делает интеграция

- **Меню** — синхронизация категорий, блюд и модификаторов из iiko в приложение
- **Заказы** — при оформлении заказа в приложении он автоматически создаётся в iiko
- **Статусы** — обновление статуса заказа в приложении при изменении в iiko (через вебхук)

## Что должен сделать партнёр (владелец заведения)

### 1. Создать API-ключ в iikoWeb

1. Войти в [iikoWeb](https://iikoweb.ru) под учётной записью заведения
2. Перейти в **Настройки** → **Настройки Cloud API** (или **Интеграции**)
3. Создать новую интеграцию:
   - Название: например, «Rest Digital»
   - Подключить нужные рестораны (организации)
   - Выбрать источник меню, типы заказов
   - Сгенерировать API-ключ (срок действия: от 2 дней до 2 лет)
4. Скопировать **API-ключ** — он понадобится в админке Rest Digital

### 2. Получить ID организации и терминальной группы

В iikoWeb или через API:

- **organizationId** — UUID организации (ресторана/точки)
- **terminalGroupId** — UUID терминальной группы (касса, куда приходят заказы)

Их можно получить через API после ввода API-ключа: в админке Rest Digital при настройке iiko будет кнопка «Загрузить конфигурацию» (или аналогичная).

### 3. Выбрать тип заказа и способ оплаты

- **orderTypeId** — UUID типа заказа (самовывоз / доставка курьером / зал)
- **paymentTypeId** — UUID способа оплаты (наличные, карта и т.п.)

Эти значения также приходят из API iiko при загрузке конфигурации.

### 4. Настроить вебхук (для синхронизации статусов)

1. В iikoWeb: **Настройки Cloud API** → **Вебхуки**
2. Указать URL: `https://ВАШ_ДОМЕН/api/webhooks/iiko`
3. Выбрать события: обновление статуса заказа
4. Сохранить

После этого при смене статуса заказа в iiko (принят, готовится, готов, выдан) статус в приложении Rest Digital будет обновляться автоматически.

### 5. Настроить Rest Digital

1. Войти в админ-панель Rest Digital (партнёрский кабинет)
2. Перейти в настройки заведения → раздел **Интеграция iiko**
3. Заполнить поля:
   - **API-ключ** — из шага 1
   - **ID организации** — из шага 2
   - **ID терминальной группы** — из шага 2
   - **ID типа заказа** — можно заполнить как общий fallback
   - **ID типа заказа DELIVERY / PICKUP / DINE_IN** — рекомендуется заполнить отдельно, если в приложении используются разные типы заказов
   - **ID способа оплаты** — из шага 3
4. Нажать **Синхронизировать меню из iiko**
5. Проверить, что меню отображается в приложении

## Требования

- Лицензия **iikoCloud** или **iikoClassic** версии 7.9.7 и выше
- Доступ к iikoWeb с правами на настройку интеграций
